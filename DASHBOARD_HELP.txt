================================================================================
 SYSTEM DIAGNOSTICS DASHBOARD — Installation & Usage Guide
================================================================================

OVERVIEW
--------
The System Diagnostics Dashboard is a web-based visual analytics tool that
lets you upload or remotely fetch daily PDF reports generated by the
system-diagnostics service and explore the data through interactive charts.

    Frontend  :  http://localhost:5000
    API server:  http://localhost:3001


PREREQUISITES
-------------
  • Windows 10 / Windows 11 (64-bit recommended)
  • Internet access (only required the first time, to download Node.js)
  • At least 500 MB of free disk space
  • PowerShell 5.1 or later (built into Windows 10/11)


QUICK START — AUTOMATED (RECOMMENDED)
--------------------------------------
The supplied PowerShell script handles everything automatically:

  1. Open the Start menu, search for "PowerShell", right-click it, and choose
     "Run as administrator" (administrator rights are needed if Node.js must
     be installed).

  2. Navigate to the repository root:

       cd C:\path\to\system-diagnostics

  3. Run the installer script:

       powershell -ExecutionPolicy Bypass -File install-and-run-dashboard.ps1

  4. Wait for the script to finish.  It will:
       a) Search for an existing npm / Node.js installation.
       b) Download and install Node.js LTS silently if npm is not found.
       c) Install dashboard dependencies (npm install).
       d) Start the dashboard server (npm run dev).

  5. Open your browser and go to:

       http://localhost:5000

  6. To stop the dashboard, press Ctrl+C in the PowerShell window.


SUBSEQUENT RUNS
---------------
After the first installation, you can start the dashboard without administrator
rights:

    cd C:\path\to\system-diagnostics
    powershell -ExecutionPolicy Bypass -File install-and-run-dashboard.ps1

The script detects that npm is already installed and skips the Node.js download
step.  It re-runs "npm install" each time (safe — it only downloads what has
changed) before starting the server.


MANUAL INSTALLATION (STEP BY STEP)
------------------------------------
If you prefer to install manually or the automated script fails, follow these
steps:

  Step 1 — Install Node.js
  ~~~~~~~~~~~~~~~~~~~~~~~~
    a. Open https://nodejs.org/ in your browser.
    b. Click the "LTS" download button (recommended for most users).
    c. Run the downloaded installer (.msi) and follow the wizard.
       Accept all defaults; make sure "Add to PATH" is checked.
    d. Open a NEW PowerShell or Command Prompt window after installation.
    e. Verify the installation:

         node --version    (should print something like v20.x.x)
         npm  --version    (should print something like 10.x.x)

  Step 2 — Install dashboard dependencies
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    a. Open PowerShell or Command Prompt.
    b. Navigate to the dashboard folder inside the repository:

         cd C:\path\to\system-diagnostics\dashboard

    c. Install dependencies:

         npm install

       This downloads all required packages into a "node_modules" folder.
       It takes 1–3 minutes depending on your internet connection.

  Step 3 — Start the dashboard
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    Remain in the "dashboard" folder and run:

         npm run dev

    You should see output similar to:

         [node]  API server running on http://localhost:3001
         [vite]  Local:   http://localhost:5000/

  Step 4 — Open the dashboard
  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    Open your browser and navigate to:

         http://localhost:5000

    To stop the server press Ctrl+C.


USING THE DASHBOARD
--------------------
  Upload PDFs (Local Mode)
  ~~~~~~~~~~~~~~~~~~~~~~~~
    Click "Upload PDF Reports" and select one or more daily_report_*.pdf
    files from your local machine.  The dashboard parses them and displays
    charts for each diagnostic run contained in the reports.

  Remote Fetch (SFTP Mode)
  ~~~~~~~~~~~~~~~~~~~~~~~~
    Click "Remote Server" and enter the SFTP connection details:
      • Host     — IP address or hostname of the Siemens server
      • Port     — SSH port (default: 22)
      • Username — your SSH username
      • Password or Private Key — authentication credentials

    Click "Test Connection" to verify connectivity and list available reports,
    then click "Fetch Reports" to download and parse them automatically.


TROUBLESHOOTING
---------------
  Problem : "npm is not recognized as an internal or external command"
  Solution: Node.js is not installed or not on your PATH.
            Re-run the PowerShell script as administrator, or install Node.js
            manually from https://nodejs.org/ and open a new terminal window.

  Problem : Script reports "execution of scripts is disabled on this system"
  Solution: Run PowerShell with the bypass flag:
              powershell -ExecutionPolicy Bypass -File install-and-run-dashboard.ps1
            Or, temporarily allow scripts for the current user:
              Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

  Problem : Port 5000 or 3001 is already in use
  Solution: Find and stop the process occupying the port:
              netstat -ano | findstr :5000
              taskkill /PID <PID> /F
            Then re-run the script.

  Problem : npm install fails with network errors
  Solution: Check your internet connection and corporate proxy settings.
            Configure npm proxy if necessary:
              npm config set proxy http://proxy.example.com:8080
              npm config set https-proxy http://proxy.example.com:8080

  Problem : Dashboard page shows a blank screen or errors in the browser
  Solution: Open the browser developer console (F12) and check for errors.
            Make sure both the API server (port 3001) and Vite (port 5000)
            are running — look for both lines in the terminal output.


UNINSTALLING
------------
  1. Delete the "node_modules" folder inside the "dashboard" directory to
     free disk space.
  2. To remove Node.js itself, go to:
       Settings → Apps → Installed Apps → search "Node.js" → Uninstall
